Applying for a new account

Vendors who can submit automatic firmware updates are in a privileged position where files can be installed on end-user systems without authentication. This means we have to do careful checks on new requests, which may take a few days to complete.

Three classes of user can be created for your organisation on the LVFS:

  • Unprivileged users that can only upload files to a private target
  • Analyst users that can view failure reports for any firmware in the OEM group
  • QA users that can upload files, and can move files from testing to stable

We can create as many different users of each type as required, and each can have a different password. Some vendors just need one 'QA User' as the person uploading the firmware is also the person who decides when to move the update from testing to stable.

If you would like to know more, or want to request a new account, please email me for more details.
